Desarrollo Web

URL:La definicion de URL segun Pérez Porto y Merino(2010) es una sigla del idioma inglés correspondiente a Uniform Resource Locator [1]
Es una direccion de internet.Donde el URL se utiliza para encontrar el sitio web y visualizarlo.Es similar a una identificacion del sitio web.

MIME: Beal define que las siglas de MIME son Multi-Purpose Internet Mail Extensions.[2]
Por lo tanto son tipos de archivos de correo electronico para leerse y ejecutarse en caso de que no esten en formato ASCII como pueden ser imagenes o videos

Metodos y codigos de errores HTTP:Los códigos de estado HTTP describen de forma abreviada la respuesta HTTP. Estos códigos están especificados por el RFC 2616.El primer dígito del código de estado especifica uno de los 5 tipos de respuesta, el mínimo para que un cliente pueda trabajar con HTTP es que reconozca estas 5 clases. La Internet Assigned Numbers Authority (IANA) mantiene el registro oficial de códigos de estado HTTP.[3] Lazaro (2018)

XHTML:Las siglas corresponden con las palabras inglesas eXtensible Hypertext Markup Language segun Alvarez (2011)[4]
Es una adaptacion del lenguaje HTML para que sea compatible con el lenguaje XML

HTML5:Perez Jimenez (2019) [5]
Define que es un estándar que sirve para definir la estructura y el contenido de una página Web.HTML 5 es la quinta version del lenguaje HTML.En esta nueva version contiene nuevos estandares y nos permite una mayor interacción entre nuestras páginas web y el contenido media así como una mayor facilidad a la hora de codificar nuestro diseño básico.

JS:Javascript es un lenguaje de programación que permite a los desarrolladores crear acciones en sus páginas web.Asi lo define Perez Valdes (2007) [6]
Es un lenguaje de programacion que programa compartamientos sobre las paginas web y responde a la interraccion del usuario.

PHP:La definicion de PHP segun Alvarez (2001) es un lenguaje para programar scripts del lado del servidor, que se incrustan dentro del código HTML. Este lenguaje es gratuito y multiplataforma.[7]
Este lenguaje se utiliza para generar paginas web dinamicas.

JSP:JSP es un acrónimo de Java Server Pages, que en castellano vendría a decir algo como Páginas de Servidor Java. Esuna tecnología orientada a crear páginas web con programación en Java segun Alvarez (2002) [8]

AJAX:Gustavo B.(2019) define que AJAX significa JavaScript asíncrono y XML (Asynchronous JavaScript and XML). Es un conjunto de técnicas de desarrollo web que permiten que las aplicaciones web funcionen de forma asíncrona, procesando cualquier solicitud al servidor en segundo plano.[9]

Preguntas a responder

¿Cuál es la diferencia entre Internet y la World Wide Web?
El internet es un sistema de computadoras que conforma una gran red de comunicación mundial y la web o World Wide Web es un sistema que funciona a través de internet, por el cual se pueden transmitir diversos tipos de datos a través del Protocolo de Transferencia de Hipertextos o HTTP.[10]

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
El propósito del protocolo HTTP es permitir la transferencia de archivos (principalmente, en formato HTML).
El método GET solicita una representación de un recurso específico. Las peticiones que usan el método GET sólo deben recuperar datos.
El método HEAD pide una respuesta idéntica a la de una petición GET, pero sin el cuerpo de la respuesta.
El método POST se utiliza para enviar una entidad a un recurso en específico, causando a menudo un cambio en el estado o efectos secundarios en el servidor.
El modo PUT reemplaza todas las representaciones actuales del recurso de destino con la carga útil de la petición.
El método DELETE borra un recurso en específico.
El método PATCH es utilizado para aplicar modificaciones parciales a un recurso. [11]

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?
POST porque cuando se utiliza este metodo se pide una respuesta que tenga en cuenta los datos proporcionados en el cuerpo de la petición HTTP[12]

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?
Con el metodo GET por que lo que hace es pasar las variables y sus valores por la URL, es decir, no solo queda a la vista de cualquier usuario, sino que además la información puede quedar guardada en el historial del navegador. [13]

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?
No ocurrio ningun error el código de respuesta 200, ok, es un código de respuesta del servidor HTTP, que nos ofrecerá el estatus ante una petición estandar correcta a la que puede responder sin problemas.[14]

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?
No,porque existe la posibilidad de que el usuario haya ingresado el URL erroneo y por lo tanto no se encontro la pagina con dicho URL erroneo.[15]

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?
Es un error que involucra un problema con el servidor.No es responsabilidad del usuario sino del desarrollador ya que no es problema del ordenador o conexion del usuario.[16]

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.
Un elemento o atributo desaprobado es aquel que ha quedado anticuado por la presencia de estructuras nuevas.Algunos atributos obsoletos son charset,name,language o nowrap.[17][23]

Sobre el ciclo de vida y desarrollo de los sistemas de información:
¿Cuál es el ciclo de vida de los sistemas de información?
El ciclo de vida de un sistema de información es un enfoque por fases del análisis y diseño que sostiene que los sistemas son desarrollados de mejor manera mediante el uso de un ciclo especifico de actividades del analista y del usuario.[22]
¿Cuál es el ciclo de desarrollo de sistemas de información?
El método del ciclo de vida para el desarrollo de sistemas consta de 6 fases:
  • Investigacion preliminar
  • Determinación de los requerimientos del sistema
  • Diseño del sistema
  • Desarrollo del software
  • Prueba de sistemas
  • Implantación y evaluación[22]
